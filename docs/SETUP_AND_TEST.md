## What Happens Behind the Scenes

### Phase 2: Preference Validation
- Validates location (maps to known cities)
- Converts budget to tier (low/medium/high)
- Normalizes cuisine names
- Ensures rating is 0-5

### Phase 3: Candidate Retrieval
- Filters by city
- Applies rating threshold
- Scores by cuisine match, budget fit, votes
- Returns top 50 scored candidates

### Phase 4: LLM Ranking
- Takes top 50 candidates
- Uses Groq LLM to rank them
- Generates personalized explanations
- Returns top K with reasoning

### Final Response
- Combines all phase outputs
- Shows phase status for debugging
- Includes costs and token usage
- Ready for user display

## Troubleshooting

### "Cannot connect to server"
- Make sure `python run_phase5_api.py` is running
- Check that port 8003 is not blocked
- Try http://127.0.0.1:8003/phase5/health in browser

### Form won't submit
- Check browser console (F12)
- Make sure you entered at least a city name
- Check network tab for failed requests

### No results or error message
- Check that all cities in Phase 3 data are supported
- Try different city spelling
- Check Phase 5 API server logs

### Slow response (>10 seconds)
- LLM first request can be slow
- Subsequent requests should be faster
- Check system resources
- This is normal for first request
